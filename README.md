# MeowMusicSync

Бот для двусторонней фоновой синхронизации любимых треков между **Spotify** и **Яндекс Музыкой**.

## Как это работает
Скрипт раз в 3 часа опрашивает обе платформы и синхронизирует новые добавленные треки. 
Используется локальный кэш в SQLite, поэтому опрашиваются только новые треки, что экономит трафик и не долбит API.
Для сопоставления названий используется умный алгоритм `fuzzywuzzy`, который умеет отбрасывать мусорные слова, но жестко штрафует, если на одной платформе это ремикс/кавер, а на другой оригинал.

## Установка и запуск

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/meowflyx/MeowMusicSync.git
   cd MeowMusicSync
   ```

2. Установите зависимости:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   # На Linux может понадобиться: sudo apt install build-essential python3-dev
   pip install -r requirements.txt
   ```

3. Переименуйте `.env.example` в `.env` и впишите свои токены:
   - `TG_BOT_TOKEN` и `TG_ADMIN_ID` — для телеграм-бота (чтобы получать уведомления об ошибках и управлять процессом).
   - `SPOTIPY_CLIENT_ID`, `SPOTIPY_CLIENT_SECRET`, `SPOTIPY_REDIRECT_URI` — ключи приложения из Spotify Developer Dashboard (редирект ставьте `http://127.0.0.1:8888/callback`).
   - `YANDEX_MUSIC_TOKEN` — токен от Яндекс Музыки (можно вытащить через расширения браузера).

4. **ВАЖНО**: Первая авторизация Spotify!
   Запустите `auth_spotify.py`:
   ```bash
   python3 auth_spotify.py
   ```
   Скрипт выдаст ссылку. Перейдите по ней в браузере, согласитесь с доступом, скопируйте URL, на который вас перекинуло (даже если там страница ошибки локалхоста), вставьте обратно в терминал и нажмите Enter. Сгенерируется файл кэша `.cache`.

5. Запуск бота:
   ```bash
   python3 main.py
   ```

## Команды бота
- `/sync_all` — Принудительно запустить полную двустороннюю синхронизацию.
- `/sync_ym_sp` — Синхронизация Яндекс -> Spotify.
- `/sync_sp_ym` — Синхронизация Spotify -> Яндекс.
- `/status` — Статистика по кэшу и маппингам.
- `/pending` — Показать треки, ожидающие одобрения.
- `/retry_failed` — Очистить кэш ненайденных треков (следующая синхронизация попробует снова).
- `/list_failed` — Список ненайденных треков.
- `/blacklist` — Показать блеклист.
- `/clear_blacklist` — Полностью очистить блеклист.
- `/unblacklist <ym_id|sp_id>` — Удалить один трек из блеклиста.
- `/clean_sp_dupes` — Удалить дубликаты из Spotify.
- `/clean_ym_dupes` — Удалить дубликаты из Яндекс Музыки.
- `/last_sync` — Информация о последней синхронизации (время и результат).
- `/logs [n]` — Последние n строк лога (по умолчанию 20, максимум 100).
- `/health` — Проверка доступности API Spotify и Яндекс Музыки.
- `/add_mapping <ym_id> <sp_id>` — Ручное сопоставление треков.
- `/help` — Список всех команд.
